Textbox System Documentation

Complete guide to action_display_text()

Click here to go back to main documentation

Basic Usage

action_display_text([
    "Hello world!"
]);

This creates a textbox using all default settings.

The text parameter is the only required parameter. Everything else is optional. This will make a basic normal text one.

Full Function Parameters

Parameter Default Description
text Required Has to be an array, even if there is only one message. Each entry in the array becomes a new page in the textbox.
snd snd_text The sound used for text blips while characters are being typed. Defaults to the standard text sound. Character-specific sounds can be used for different voices.
on_bottom false Determines whether the textbox appears at the bottom of the screen. Useful when a character or important object would otherwise be covered by the textbox.
type_speed 1 Controls how quickly characters appear on screen. Higher values result in slower text scrolling. Faster than 1 text is not implemented yet, but is planned
next_message_timer 90 Time in frames before automatically advancing to the next page. Primarily used for automatic dialogue sequences. (60 frames = 1 second)
has_face false Enables the portrait system. Must be set to true for portraits to appear.
face_sprite -1 The sprite containing all portrait expressions for the speaking character. Use an array if multiple are used (For example Susie and Ralsei talking to one another). You must link the array to message number, so if one actor speaks twice, then the other actor speaks it would be [actor_a_sprite, actor_a_sprite, actor_b_sprite]
face_id -1 Determines which portrait frame is displayed. Can be a single number for all pages, or an array to change expressions between pages. Same array rules from above apply here.
spacing 2 Additional spacing between characters. Useful for custom fonts, improving readability, or those spooky long text moments. Same array rules from above apply here.
change_sprite_when_talking false When enabled, the speaker's overworld sprite will animate while text is being typed. Same array rules from above apply here.
actor -1 The actor ID of the character speaking. Required when using talking animations. Same array rules from above apply here.
overworld_sprite_arr -1 Array of sprites used while the actor is talking. The textbox system cycles through these sprites during dialogue. Same array rules from above apply here.
sprite_spd_arr -1 Animation speeds corresponding to each sprite in overworld_sprite_arr. Same array rules from above apply here.

Portrait System

has_face

action_display_text(
    ["Hello"],
    snd_text_susie,
    false,
    1,
    120,
    true
);

Enables portrait rendering.

face_sprite

spr_face_susie

Sprite containing all portrait frames.

face_id

The most important portrait parameter.

Single Portrait

action_display_text(
    ["Hello"],
    snd_text_susie,
    false,
    1,
    120,
    true,
    spr_face_susie,
    3
);

Uses frame 3 of spr_face_susie for all pages.

Portrait Per Page

action_display_text(
    [
        "Hello.",
        "Wait...",
        "WHAT?!"
    ],
    snd_text_susie,
    false,
    1,
    120,
    true,
    spr_face_susie,
    [10,4,6]
);
PagePortrait Frame
Hello.10
Wait...4
WHAT?!6
This allows portraits to change emotion during a conversation.

pacing

action_display_text(
    ["Hello"],
    snd_default_text,
    false,
    1,
    120,
    false,
    -1,
    -1,
    5
);

Controls additional spacing between letters. Not suggested to change from 2 unless you know what you are doing

Talking Animation System

change_sprite_when_talking

true

When enabled, the overworld actor animates while speaking.

actor

"Susie"

The actor_id to animate.

overworld_sprite_arr

[
    spr_susie_walk_right,
    spr_susie_laugh,
    spr_susie_walk_up
]

Sprites cycled while talking.

sprite_spd_arr

[
    0, // Not walking
    3, // For a very fast laugh idk
    1, // Walking while standing still
]

Animation speed for each talking sprite.
